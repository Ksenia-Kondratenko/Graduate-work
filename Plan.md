1. Перечень автоматизируемых сценариев:
    а) Позитивные сценарии: 
оплата картой: пользователь вводит валидные данные карты, проходит успешную оплату через Payment Gate; 
получение кредита: пользователь запрашивает кредит, используя свои банковские данные, операция одобряется через Credit Gate;
проверка сохранения информации о платежах: проверяется наличие записи в базе данных о проведённом платеже по каждому из способов оплаты.
    б) Негативные сценарии:
ошибка платежа: оплата отклоняется сервисом Payment Gate из-за неверных данных карты;
отказ в кредите: запрашиваемый кредит отклоняется сервисом Credit Gate;
запрет хранения данных карты: после успешной операции проверяется отсутствие сохранённых данных карты в базе данных.

2. Перечень используемых инструментов:
Python 3 - удобный для написания тестов язык программирования благодаря широкому спектру готовых библиотек;
Requests - библиотека, которая упрощает взаимодействие с HTTP(S)-API. Используется для отправки GET, POST-запросов и удобна для тестирования RESTful-сервисов, включая эмулируемые шлюзы банка и кредитной организации;
Pytest - удобный фреймворк для модульного и функционального тестирования на Python. Позволяет писать читаемые и поддерживаемые тесты, интегрироваться с другими инструментами и расширять возможности посредством плагинов;
Node.js - среда,  которая используется для разработки серверных приложений и скриптов, написанных на JavaScript;
Selenium WebDriver - автоматизирует работу браузера, позволяет проводить тесты над элементами интерфейса;
Allure - используется для формирования удобных отчётов по результатам тестирования, отображает ход прохождения тестов, помогает быстро выявить проблемные места;
Docker Compose - удобен для разворачивания инфраструктуры приложения и настройки среды тестирования;
Postman: используется для проверки API интеграций с сервисами (Payment Gate и Credit Gate);
GitHub Actions - инструмент для настройки непрерывной интеграции и доставки (CI/CD). Они помогают автоматически запускать тесты при каждом коммите или изменении веток, обеспечивают быстрое получение обратной связи.

3. Возможные риски при автоматизации:
- технические ошибки: например, проблемы с сетью, которые могут вызвать ложноположительные сбои тестов;
- неправильная конфигурация окружения: ошибки конфигурации контейнеров Docker могут препятствовать запуску всех компонентов приложения;
- ошибки настройки баз данных: неправильная настройка учетных записей или URL-соединений к базе данных приведет к проблемам с сохранением данных; 
- обновления зависимых библиотек: регулярные обновления зависимостей могут вызвать конфликты версий и проблемы совместимости;
- изменения API: если спецификации или поведение сторонних сервисов изменится, возможно появление ошибок интеграции, требующих обновления теста.
