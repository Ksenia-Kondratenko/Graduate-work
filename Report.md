# Отчёт о проведённом тестировании

1. Краткое описание.

В рамках данного проекта проведено автоматизированное тестирование приложения оплаты тура с использованием Selenium. Были проверены основные позитивные и негативные сценарии использования приложения при оплате тура без кредита, оплате тура с использованием кредита, а также взаимодействие с базой данных.

2. Количество тест-кейсов: 136


3. Процент успешных и неуспешных тест-кейсов:
- Успешных: 77,94% (106)
- Неуспешных: 13,97% (19).
Основные ошибки связаны с отсутствием таблицы в базе данных, несоответствием ожидаемого текста ошибки и фактического, а также ошибки в основной логике приложения - проведении оплаты.
- С ошибкой: 8,08% (11).
Большинство ошибок в тестах связаны с превышением времени ожидания элементов, несоответствующими селекторами и некорректными таймингами.
 

4. Общие рекомендации:
- Уточнить корректность XPath и ожиданий в тестах.
- Повысить производительность страницы, если наблюдаются длительные ожидания.
- Проверить миграцию данных и логику создания таблиц в приложении.
